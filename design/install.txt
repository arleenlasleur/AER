Описание статистики, соглашение и доступ к ней.

Реальность современного интернета - жесточайший кризис IPv4-адресов. На одном внешнем IP может сидеть целый дом, квартал и
даже микрорайон. А сервера для обслуживания статистики стоят денег. Поэтому, для размещения сервиса используется оверлейная
сеть I2P. Она полностью анонимна, так что из вашей статистики никто не сможет извлечь персональных данных. Мы используем
роутер i2pd - форк i2p-роутера, переписанный на C++. У него меньше системные требования и выше производительность.

Для доступа к статистике приложен набор конфигурации i2pd - файлы i2pd.conf и tunnels.conf

*** Сам i2pd в комплект не входит, чтобы в архиве не было исполняемых файлов ***

Вы сами решаете, скачивать, запускать i2pd или нет.

Установка.

Сборки i2pd для Windows находятся на https://github.com/PurpleI2P/i2pd/releases
Полный список можно посмотреть по ссылке Show all assets.

1. В зависимости от системы, выберите файл, имя которого заканчивается на _win32_mingw.zip, _win64_mingw.zip или _winxp_mingw.zip
   i2pd будет работать в портативном режиме и не нуждается в административных привилегиях, можно разместить его внутри каталога unreal.

2. Распакуйте из архива файл i2pd.exe в нужный каталог, например D:\unreal\i2pd
   Распакуйте каталоги webconsole и certificates в тот же каталог (то есть в целевом каталоге они должны быть вне каталога contrib,
   рядом с файлом i2pd.exe)

3. Убедитесь, что прилагаемые файлы конфигурации (i2pd.conf и tunnels.conf) находятся рядом с файлом i2pd.exe
   Следует использовать эти файлы из нашего архива, а не из дистрибутива i2pd (в нём приложены настройки по умлочанию).

4. Запустите файл i2pd.exe
   Значок i2pd будет в systray. Также, будет доступна его веб-консоль по адресу http://127.0.0.1:7070

5. Если похоже на то, что i2pd недоступны соединения: разрешите процессу i2pd.exe доступ. Для сторонних брандмауэров способ сделать
   это, может отличаться. Для стандартного Windows Firewall, это делается в окне настройки правил исходящих соединений. Чтобы
   открыть его, следуйте: Панель управления -> Windows Firewall -> Advanced settings -> Inbound rules.
   Обычно, Windows Firewall сразу показывает окно предупреждения о том, что процессу требуется доступ в сеть. В этом окне нужно
   просто нажать кнопку Allow/Разрешить доступ. Правило при этом будет создано автоматически.
   
   Альтернативный способ:
   - запустите cmd или powershell с административными привилегиями, например при помощи команды
     powershell -command "start-process cmd -verb runas"
   - введите там команду
     netsh advfirewall firewall add rule name="i2pd" dir=in action=allow program="D:\unreal\i2pd\i2pd.exe" enable=yes
     где D:\unreal\i2pd - путь к каталогу i2pd.

   Критерии того, что i2pd работает нормально, можно отследить на странице веб-консоли:
   - Routers - больше 0
   - Floodfills - меньше чем Routers, но больше 0
   - Client tunnels - больше 2
   - Tunnel creation success rate - лучше 0%
   - увеличиваются счётчики трафика (Received, Sent)

Правильная остановка i2pd.
- из веб-консоли: Router commands -> Start graceful shutdown
- из меню значка в systray: Start graceful shutdown
i2pdне будет открывать новые туннели, и через 10 минут все клиенты отсоединятся. Если просто снять процесс i2pd, это создаст
внезапный обрыв туннелей - такие ситуации ухудшают качество связи по I2P. Старайтесь выгружать процесс только по graceful shutdown.
